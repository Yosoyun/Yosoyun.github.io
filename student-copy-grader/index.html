Quality OCR Integration

The best practical setup is Mathpix for handwritten STEM OCR, then OpenAI vision/reasoning for rubric grading and deduction reasons. Browser OCR remains available only as a free fallback.

Implementation Order

Step 1: Deploy the backend in `student-copy-grader/backend` with `MATHPIX_APP_ID`, `MATHPIX_APP_KEY`, and `OPENAI_API_KEY`.
Step 2: Paste the backend URL in OCR Mode and choose Quality backend OCR.
Step 3: Upload a 10-copy pilot batch and compare against teacher marks before using it for routine checking.
Step 4: Keep the teacher approval screen as the final authority. AI should assist, not silently finalize.

Complete teacher guide

How To Use Student Copy Grader

This app helps teachers check answer copies faster. It reads question papers, solution keys, and student answers locally in the browser, then applies a marking rubric and shows question-wise marks with deduction reasons.

Try Demo

Click Load Demo, then Grade. It loads a Class XII Mathematics paper, solution key, rubric, and a student answer written in a different style.

1. Load Paper

Paste the question paper text or attach a TXT, Markdown, CSV, JSON, PDF, screenshot, or scanned image. Click Read File to extract text.

2. Load Key

Import a rubric JSON for best results, or upload/paste a plain solution key. Click Draft Rubric to create an editable first marking scheme.

3. Save Setup

After the paper and key are ready, click Save Setup. For the next student, click New Student and upload only that student's copy.

4. Add Student

Enter student ID and name. Upload answer JSON, text, PDF, screenshot, or scanned copy. Click Read Copy for OCR when needed.

5. Review OCR

OCR is helpful, not final. Check the extracted answers, especially handwriting, formulas, diagrams, and crossed-out work.

6. Grade

Click Grade. The app gives marks, confidence, and feedback. Any unclear or partial answer is marked for teacher review.

7. Save & Export

Save the student result, export class CSV, download JSON backup, or download an HTML report for sharing.

How Grading Works

MCQ: The selected option is normalized, so `B`, `Option B`, and `Answer: B` can match the same key.
Numeric: The app checks every number written in the answer, so working like `1×4 - 2×3 = -2` can still match `-2`.
Step marking: A rubric can split marks into value-points, such as substitution, integration, final answer, point, slope, and tangent equation.
Different wording: Each value-point can use accepted phrases, keywords, and a reference-answer similarity check. This catches many alternate student explanations.
Your JSON: The app now accepts `qno`, `keywords`, `alternateAnswers`, `correct_option`, `final_answer`, and `subparts`, then converts them into the internal rubric automatically.
Teacher review: Partial marks, low confidence, OCR uncertainty, formulas, and unusual methods should be reviewed before final marks are saved.

Supported Files

Best format: Rubric JSON and student answers JSON. This gives the most reliable marking.
Readable text: TXT, Markdown, CSV, and JSON are read directly in the browser.
PDF: PDF.js extracts text from normal PDFs. If a PDF is scanned, the app can OCR rendered pages.
Images and screenshots: Tesseract.js OCR reads PNG, JPG, and other browser-supported image files.
Teacher decision: Final marks should be approved by a teacher. The app should not be treated as an autonomous board examiner.

Project Report

Student Copy Grader is an open, free, local-first project for helping teachers check descriptive student answers against a question paper and marking scheme.

Demand

Teachers often check 40 or more handwritten copies per test. The pain is strongest when answers are descriptive, stepwise, or CBSE-style, because students and parents also need a clear reason for every deduction.

Supply Gap

Commercial AI grading products exist, but many are paid, cloud-based, or institution-priced. Open-source OCR and grading models exist separately. This project joins the useful pieces into a teacher-facing workflow.

Current Website Scope

  • Question paper and marking scheme workspace.
  • Rubric JSON validation.
  • MCQ, numeric, and value-point short-answer grading.
  • Question-wise feedback and review flags.
  • Browser-local records and exports.

Open Source Engines

  • PDF.js reads text from normal PDFs and renders scanned PDF pages for OCR.
  • Tesseract.js performs browser-side OCR for screenshots, images, and scanned pages.
  • The rubric checker is local JavaScript in this project, with no paid API call.

Handwriting OCR Roadmap

The static website version does not claim perfect handwritten OCR. The next offline layer should connect OCR engines such as PaddleOCR, docTR, TrOCR, Tesseract, and math OCR tools such as pix2tex or Pix2Text. The teacher should verify OCR before final marks are locked.

Evaluation Plan

  • Compare OCR text against human-corrected text.
  • Compare marks against teacher-approved marks.
  • Track exact agreement, within-one-mark agreement, and teacher override rate.
  • Measure minutes saved per copy.
